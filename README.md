## About This Portfolio

The SOPs in this repository are **illustrative analogues** of proprietary documentation I authored or edited during a Product Lifecycle Management (PLM) system migration at a regulated medical device manufacturer.

The originals supported a department-wide transition from a legacy PLM platform to an enterprise-grade replacement. Because those documents are proprietary, I have recreated them here using:

- A fictional company: **Meridian Medical**
- A fictional legacy system: **LegacyVault PLM**
- A fictional replacement system: **Enterprise CorePLM**

The structure, scope, and writing approach of each SOP faithfully reflect the originals. Subject matter has been generalized to remove proprietary content while preserving the documentation patterns, voice, and cross-referencing conventions I applied in the actual work.

---

## Guiding Principles

These principles governed the writing of every document in this set.

**1. Consistent terms, not synonyms.**  
Each concept has one name. That name is used every time. Synonyms introduce ambiguity and force readers to decide whether two terms mean the same thing.

**2. Clarity, not condescension.**  
These documents are written for experienced professionals learning a new system — not beginners learning a domain. The goal is to bring the system to the user, not to explain work they already know how to do.

**3. Cross-references, not double-documenting.**  
A process appears in exactly one place. Other documents link to it. Duplicating content across documents makes the set impossible to maintain.

**4. Global English.**  
Language is precise and unambiguous for both native and non-native speakers. Sentences are short. Active voice is preferred. Idioms and unnecessarily complex constructions are avoided.

**5. Happy Path priority.**  
Each SOP documents the standard workflow clearly. Exceptions, errors, and edge cases are addressed in [SOP-007: Troubleshooting](./SOP-007-troubleshooting.md) rather than interrupting the main procedure.

## Document Conventions

| Convention | Meaning |
|------------|---------|
| **Bold text** | UI element, field name, or button label |
| `Monospace text` | System-generated value or exact string to enter |
| > Note: | Supplementary information |
